Configure how delivery bookings are routed based on consignee and transport mode.

How Delivery Booking Rules work:

  • This rule is for FCL deliveries only.
  • Customers choose suitable booking dates.
  • Ligentia users see a warning but can select any date.
  • "Minimum Book (Days)": Earliest date for customer selection from today, allowing haulier booking.
  • "Book From": Earliest date after ETA for delivery selection, considering unloading, customs, and transportation.
  • Each rule defines when TOPS auto-booking or email routing applies for a given customer, lane and load type.
  • Cut-off times apply to amendments and cancellations, not booking creation.
  • Rules are matched by specificity: Country → Port → MOT → Carrier → Load. The most specific matching rule wins. If no rule matches, the booking is held and a System Alert is raised.

Configure how delivery bookings are routed based on consignee and transport mode.

How Delivery Booking Rules work:

  • Routing: Consignments are matched against rules from most specific (POD + Mode) to most general. Rules are processed in sequence. If a shipment matches multiple criteria, the most specific port-level rule will take precedence. Choosing Email as the method allows you to specify a custom list of internal and external stakeholders who will receive a delivery booking notification each time a new booking is created in the Delivery Module.
  • TOPS Integration: Automated system-to-system booking. Only available for supported UK ports and FCL mode.
  • Email Fallback: For manual handling. Multiple email addresses can be added (separated by commas).
