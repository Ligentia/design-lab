ⓘ How this page works Click to expand ▾
①
First Approval (Auto-approval)
When a supplier submits a booking, each active rule is evaluated against every purchase order line. If all rules pass, the PO is automatically approved — no manual action needed. If any rule fails, the PO moves to the manual queue for a merchandiser or buyer to review.

Rules only run when First Approval is ON. If First Approval is turned off globally, all bookings go straight to manual review — but Highlight Validation still applies to help approvers see why a booking needs attention.
②
Highlight Validation
Validation styling highlights failing rule conditions directly on the booking, even when a PO is in manual review. For example, a quantity mismatch shows a red cell on the quantity column; a wrong POL highlights the port field.

This works independently of First Approval — you can run a rule in styling-only mode (V badge) so it never auto-approves but always flags the issue visually. Useful for clients who want full manual control but still want structured highlights.

Rules marked N/A for Highlight Validation bypass this entirely (e.g. trusted supplier rules don't have a field to highlight).
③
Review Hold
Some bookings require a second layer of sign-off from the Shipping or Senior team — even if all POs are approved by the merchandiser. When Review Hold is ON for a rule, and that rule's condition is present on the booking (e.g. container type is GOH, freight terms are Air Collect, CBM is below minimum), the entire booking is held after PO-level actioning.

Key behaviour: At least one PO must be approved — a fully rejected booking does not proceed to review hold. The Shipping team see the booking in their queue with the reason clearly displayed. They can approve or reject the booking in full.

Review Hold works independently of First Approval — a booking can be manually approved at PO level and still trigger a review hold at booking level.

First Approval toggle
Controls whether this rule contributes to the auto-approval decision for each PO. When OFF, the rule is skipped entirely for approval purposes — only Highlight Validation runs if that toggle is separately ON.
⚠️ Turning a rule OFF does not remove it from the system — it is simply skipped. Settings are preserved.
Review Hold toggle
When ON, if the rule's condition is detected on the booking, the booking is held for Shipping team review after all POs are actioned. The approver sees the reason, rule name, and field value in their queue.
Not all rules support Review Hold — rules that operate at PO level only (e.g. quantity rules) show No and cannot be changed.

⚠️ Important — before enabling rules
Data dependencies
Every rule depends on specific data fields being present on the PO feed. Enabling a rule without the required data will cause false failures or the rule being silently skipped. Always verify the client's PO feed mapping before activating rules — particularly date tolerance rules (require Exfactory / CRD fields), SA Validation (requires Production Approval mapping), and quantity rules (require ordered quantities).
Rule interactions — AND logic
A PO must pass every active First Approval rule to auto-approve. Rules use AND logic — there is no partial pass. If 5 rules are enabled and a booking passes 4 but fails 1, the PO goes to the manual queue.

Example: Booked MOT = Sea (passes ✓) + Quantity Tolerance ±5% with a 10% variance (fails ✗) → PO goes to merchandiser queue, even though MOT was correct.

The only exception is Trusted Supplier: if the booking supplier is on the trusted list, all other rules are bypassed and the PO auto-approves immediately. Review Hold always runs at booking level after PO decisions are complete, regardless of how POs were actioned.
Full approval flow
1. Supplier submits booking → 2. Rules evaluate each PO → 3a. All pass → Auto-approved / 3b. Any fail → Merchandiser queue → 4. Merchandiser approves/rejects each PO → 5. All POs actioned → 6. Review Hold triggered? → Shipping team queue → 7. Booking fully resolved
Auto-approval OFF globally: Steps 3a never applies — all bookings go to step 3b. Rules still run to power Highlight Validation at step 4.
Fully rejected booking: If all POs are rejected at step 4, the booking is closed — Review Hold at step 6 does not apply.
Mixed result: If some POs are approved and some rejected, Review Hold still applies if triggered — the Shipping team reviews the booking with all its POs.
⚠ Rules use AND logic — all active rules must pass for a PO to auto-approve. If you have 5 rules enabled and a booking passes 4 but fails 1, the PO goes to the merchandiser queue. For example: if Booked MOT is set to Sea and Quantity Tolerance is set to ±5%, a booking that arrives with the correct MOT but a quantity 10% over will fail the quantity rule and go to manual review — even though MOT passed. Every rule you enable is an additional gate that every booking must clear.

⚠ Not all rules will work for every client. Each rule relies on specific data being present on the PO feed. Date tolerance rules require Exfactory / CRD fields; SA Validation requires a Production Approval field mapping; quantity rules require ordered quantities. Before enabling a rule, confirm the required data is available for this client. If unsure, contact the Product team before applying rules.
